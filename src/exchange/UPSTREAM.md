## PRESERVE list

- **`StartupLogger.cs`** (Phase 00; namespace `Bifrost.Exchange`) ‚Äî writes `/tmp/bifrost-ready` sentinel consumed by the Dockerfile HEALTHCHECK. Registered as a `BackgroundService` alongside `CommandConsumerService` ‚Äî the two hosted services are intentionally decoupled: StartupLogger signals "process booted + DI graph resolved", CommandConsumerService signals "RabbitMQ topology declared + poll loop running". The sentinel fires before the poll loop's first iteration; this is acceptable for a LAN smoke gate.
- **DI graph ordering** ‚Äî matches Arena's Program.cs: connection ‚Üí clock ‚Üí instruments/engines ‚Üí publisher stack ‚Üí rules ‚Üí RoundState ‚Üí validator + publishers + service ‚Üí hosted services. The ordering is semantically equivalent to Arena's (Arena had an additional telemetry sub-tier between "rules" and "validator"); BIFROST collapses that tier.
- **Host style** ‚Äî `Host.CreateApplicationBuilder(args)` (generic worker host). Top-level statements, no `class Program` wrapper, matching Phase 00 precedent per STATE.md (worker Program.cs is top-level-statements style per PATTERNS.md C2).
