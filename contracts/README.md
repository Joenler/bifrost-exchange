# BIFROST gRPC wire contracts

Authoritative `.proto` files for the team-facing gRPC surface. Six files:
`market.proto`, `strategy.proto`, `auction.proto`, `round.proto`,
`events.proto`, `mc.proto`.

## Ground rules

1. **Codegen is not done by `buf`.** `buf` is used for lint + breaking-change
   gating only. .NET code is generated by `Grpc.Tools` at MSBuild time;
   Python code is generated by `grpcio-tools` (wheel build + CI). Do not
   ship a `buf.gen.yaml`.
2. **No `google.protobuf.Timestamp`.** All timestamps are `int64 <name>_ns`
   (nanoseconds since Unix epoch). The JSON-string-nanoseconds representation
   used by the internal RabbitMQ DTOs never reaches the gRPC wire.
3. **Every enum has `<PREFIX>_UNSPECIFIED = 0`** as its first value. This
   satisfies `buf lint` DEFAULT tier and aligns with protobuf's
   default-value-is-zero semantics.
4. **No `google.protobuf.Any`.** MC command payloads stay typed via the
   `McCommand` oneof.

## Layout

```
contracts/
  market.proto       # shared: Instrument, Side, OrderType, BookView, BookLevel
  strategy.proto     # team-facing service + StrategyCommand/MarketEvent oneofs
  auction.proto      # BidMatrix + ClearingResult (DAH)
  round.proto        # State enum + RoundState
  events.proto       # Event wrapper + Regime/Severity enums + payloads
  mc.proto           # McCommand (21 oneof variants) + McCommandResult
  buf.yaml           # lint + breaking-change config (added later)
  dotnet/            # per-surface NuGet csprojs (added later)
  python/            # umbrella wheel manifest (added later)
  roundtrip/         # polyglot round-trip harness (added later)
```

## Upstream provenance

Proto style (`package <name>.v1;`, `option csharp_namespace`,
`<PREFIX>_UNSPECIFIED = 0` enum values, scaled-integer `*_ticks` fields)
carries over from Arena's `protos/pricing.proto`. BIFROST deviates in one
place: Arena uses `google.protobuf.Timestamp`; BIFROST uses raw `int64
*_ns` fields.
